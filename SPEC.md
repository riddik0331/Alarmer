# Спецификация проекта «Будильник»

## 1. Название проекта

**Будильник** — десктопное приложение-будильник для Windows с современным тёмным интерфейсом, поддержкой системного трея, кастомных звуков и постепенного увеличения громкости.

---

## 2. Цель проекта

Разработать полнофункциональное десктопное приложение-будильник для Windows, которое позволяет:

- Создавать неограниченное количество будильников с гибкой настройкой (дни недели, время, звук, громкость, метка).
- Надёжно срабатывать в заданное время даже при свёрнутом в трей окне.
- Воспроизводить звуковой сигнал с возможностью отложить или выключить.
- Сохранять все настройки между сеансами работы.
- Иметь современный привлекательный дизайн с анимациями.

**Целевая аудитория:** Пользователи Windows, которым нужен простой, но стильный десктопный будильник с возможностью тонкой настройки.

## 4. Функциональные требования

### 4.1. Главный экран — Список будильников

**Назначение:** Отображение всех созданных будильников в виде карточек.

**Элементы:**

| Элемент                 | Описание                                                     |
|-------------------------|--------------------------------------------------------------|
| **Заголовок**           | «Будильники» (слева), кнопка «+» (справа) для добавления нового |
| **Карточка будильника** | Блок с информацией об одном будильнике (см. ниже)            |
| **Пустое состояние**    | Если нет ни одного будильника — отображается заглушка: «Нет будильников. Нажмите +, чтобы добавить» |

**Состав карточки будильника:**

| Поле               | Тип отображения                                   | Поведение                                         |
|--------------------|---------------------------------------------------|---------------------------------------------------|
| **Время**          | Крупный текст «07:30»                             | Всегда видно                                      |
| **Дни недели**     | Строка «Пн Вт Ср Чт Пт Сб Вс» (активные выделены) или текст «Единоразово» | Под временем |
| **Название (метка)** | Текст (например, «Подъём»)                      | Если не задано — скрывается или показывается плейсхолдер |
| **Toggle вкл/выкл** | Переключатель (QCheckBox в стиле switch)          | Меняет состояние enabled/disabled (см. 4.1.1)     |
| **Кнопка удаления**  | Иконка корзины 🗑 (стилизованная)                | Удаляет будильник после подтверждения              |

**Требования к карточкам:**

- Карточки отображаются вертикальным списком с отступами.
- Каждая карточка имеет тень (drop shadow) и скруглённые углы.
- Выключенные будильники визуально затемнены (opacity ~0.5) и имеют пометку «Выкл».

#### 4.1.1. Toggle вкл/выкл

- При переключении в **выкл** — будильник не срабатывает, но остаётся в списке.
- При переключении в **вкл** — будильник активируется.
- Состояние сохраняется немедленно в JSON.
- Если будильник был единоразовым и уже сработал — он автоматически переводится в «выкл» (или помечается как «исполнен»).

#### 4.1.2. Кнопка удаления

- При нажатии появляется диалог подтверждения: «Удалить будильник "Название" (07:30)?».
- После подтверждения будильник удаляется из списка и из JSON.
- Можно отменить удаление.

### 4.2. Экран создания / редактирования будильника

**Назначение:** Создание нового или изменение существующего будильника.

**Открывается:**

- По кнопке «+» на главном экране — создание нового.
- По клику на карточку будильника — редактирование существующего.

**Форма содержит:**

#### 4.2.1. Выбор времени

| Поле               | Элемент UI                            | Валидация          |
|--------------------|---------------------------------------|--------------------|
| Часы (00–23)       | QSpinBox (с кнопками вверх/вниз)      | 00–23, шаг 1       |
| Минуты (00–59)     | QSpinBox (с кнопками вверх/вниз)      | 00–59, шаг 1       |

- Альтернативно: QTimeEdit с маской ЧЧ:ММ.
- Текущее выбранное время должно быть хорошо читаемо (крупный шрифт).

#### 4.2.2. Дни недели

- **7 чекбоксов** в ряд: «Пн», «Вт», «Ср», «Чт», «Пт», «Сб», «Вс».
- **Чекбокс «Единоразово»**: при его активации все дни недели деактивируются (и наоборот).
- Если выбран хотя бы один день недели, «Единоразово» автоматически снимается.
- По умолчанию: «Единоразово» включён.

#### 4.2.3. Выбор звука

**Встроенные звуки (3–5 предустановленных):**

1. «Классический» (звонок)
2. «Нежный» (колыбельная / мягкий сигнал)
3. «Природа» (птицы / ручей)
4. «Энергичный» (рок / энергичная мелодия)
5. «Лаунж» (спокойная музыка)

- Отображаются в QComboBox для выбора.
- Кнопка **«Прослушать»** (Play) рядом с комбобоксом — воспроизводит выбранный звук.
- Кнопка **«Стоп»** (Stop) — останавливает предпрослушивание.

**Пользовательский файл:**

- Кнопка **«Выбрать файл…»** — открывает QFileDialog с фильтром: `*.wav *.mp3`.
- После выбора отображается полный или сокращённый путь к файлу.
- Если выбран пользовательский файл — встроенные звуки игнорируются.
- Кнопка **«Сбросить»** (справа от пути) — возвращает к встроенному звуку по умолчанию.

#### 4.2.4. Регулировка громкости

- **QSlider** (горизонтальный) с диапазоном 0–100.
- Визуальная метка процентов (например, «75%») рядом или над слайдером.
- Иконка динамика слева от слайдера (меняется: выкл / тихо / громко).
- По умолчанию: **80%**.

#### 4.2.5. Название будильника (метка)

- **QLineEdit** с плейсхолдером «Название будильника…».
- Максимум 50 символов.
- Необязательное поле.

#### 4.2.6. Постепенное увеличение громкости

- **QCheckBox** «Плавно увеличивать громкость» (или «Fade-in»).
- Если включён — при срабатывании будильника громкость плавно растёт с 0% до заданного уровня за **30 секунд** (см. 4.3.4).
- По умолчанию: **выключен**.

#### 4.2.7. Кнопки управления формой

| Кнопка     | Действие                                                      |
|------------|---------------------------------------------------------------|
| «Сохранить» | Сохраняет будильник в JSON, возврат на главный экран          |
| «Отмена»    | Возврат на главный экран без сохранения                       |

- Валидация: если время не указано — кнопка «Сохранить» неактивна (disabled).

### 4.3. Срабатывание будильника

#### 4.3.1. Триггер срабатывания

- Фоновый таймер (QTimer) проверяет каждые **30 секунд** соответствие времени активных будильников системному времени.
- При совпадении: будильник срабатывает.

#### 4.3.2. Окно срабатывания (Alarm Window)

- **Окно поверх всех окон** (WindowStaysOnTopHint).
- Размер: 400×300 px (адаптивно).
- Отображается даже если приложение свёрнуто в трей.
- **Тёмный фон** с эффектом пульсации (см. 4.3.5).

**Элементы окна:**

| Элемент                        | Описание                                                          |
|--------------------------------|-------------------------------------------------------------------|
| **Заголовок**                  | Название будильника (или «Будильник», если метка не задана)       |
| **Текущее время**              | Крупные часы (например, «07:30»)                                  |
| **Кнопка «Выключить»**         | Большая круглая красная кнопка                                    |
| **Кнопка «Отложить на 5 мин»** | Средняя кнопка, при нажатии — звук отключается, будильник срабатывает через 5 мин |
| **Кнопка «Отложить на 10 мин»**| Аналогично                                                        |
| **Кнопка «Отложить на 15 мин»**| Аналогично                                                        |

- Если единоразовый будильник — после «Выключить» он автоматически деактивируется (состояние enabled = false) или удаляется (настраивается).
- Если повторяющийся — после «Выключить» будильник остаётся активным до следующего дня.

#### 4.3.3. Воспроизведение звука

- Используется **QMediaPlayer** для MP3/WAV.
- Звук воспроизводится циклично (loop), пока не нажата кнопка «Выключить» или «Отложить».
- При «Отложить» — звук останавливается; при следующем срабатывании начинается заново.
- Если fade-in включён — громкость стартует с 0 и плавно растёт (см. 4.3.4).

#### 4.3.4. Постепенное увеличение громкости (Fade-in)

- Длительность: **30 секунд**.
- Стартовая громкость: 0%.
- Конечная громкость: заданный пользователем уровень.
- Шаг увеличения: каждые 200 мс громкость увеличивается на (target_level / 150) процентов.
- Fade-in работает только для данного будильника, не влияет на системную громкость.

#### 4.3.5. Визуальная анимация привлечения внимания

- **Пульсация фона** (через QPropertyAnimation):
  - Цвет фона плавно меняется между #1a1a2e и #2d004d (тёмно-красный/фиолетовый).
  - Период: 1.5 секунды.
- **Мигание заголовка**: текст заголовка плавно меняет прозрачность (0.3 → 1.0 → 0.3).
- Окно появляется с анимацией scale (увеличение от 0.8 до 1.0 за 300 мс).

### 4.4. Системный трей

#### 4.4.1. Поведение при закрытии окна

- При нажатии на крестик (Close) главного окна — приложение **не завершается**.
- Окно скрывается (hide), в трее появляется иконка.
- Завершить приложение можно только через контекстное меню трея «Выход» (или через диспетчер задач).

#### 4.4.2. Иконка в трее

- Кастомная PNG/SVG иконка (колокольчик).
- При активных будильниках — цветная.
- Если все будильники выключены — серая/полупрозрачная.

#### 4.4.3. Контекстное меню трея

| Пункт          | Действие                                                     |
|----------------|--------------------------------------------------------------|
| «Показать»     | Разворачивает главное окно (show, raise)                     |
| «Настройки»    | Открывает главное окно (если скрыто) и фокусирует его        |
| «Выход»        | Завершает приложение (QApplication.quit)                     |

#### 4.4.4. Уведомления (Toast)

- При срабатывании будильника (если окно приложения свёрнуто или скрыто) — показывается системное уведомление Windows.
- Текст: «🔔 [Название будильника] — 07:30».
- Клик по уведомлению открывает главное окно.

### 4.5. Сохранение и загрузка данных

#### 4.5.1. Формат хранения

- **JSON-файл** `alarms.json`.
- Путь:
  - В режиме разработки: в директории проекта (`./alarms.json`).
  - В собранном .exe: в `%APPDATA%/Budilnik/alarms.json` (через `os.path.expandvars('%APPDATA%')` или `QStandardPaths.writableLocation(QStandardPaths.AppDataLocation)`).
- Кодировка: UTF-8.

#### 4.5.2. Структура JSON

```json
{
  "version": 1,
  "alarms": [
    {
      "id": "uuid-ст-рока",
      "enabled": true,
      "title": "Подъём",
      "time": "07:30",
      "days": [1, 2, 3, 4, 5],
      "once": false,
      "sound_source": "builtin",
      "sound_name": "classic",
      "sound_file": null,
      "volume": 80,
      "fade_in": false,
      "snoozed_until": null
    }
  ]
}
```

**Пояснение полей:**

| Поле           | Тип      | Описание                                                |
|----------------|----------|---------------------------------------------------------|
| `id`           | string   | UUID v4 (генерируется через `uuid` или `uuid4()`)       |
| `enabled`      | bool     | Активен ли будильник                                    |
| `title`        | string   | Название (может быть пустым `""`)                       |
| `time`         | string   | Время в формате "ЧЧ:ММ"                                 |
| `days`         | list[int]| Дни недели (1=Пн … 7=Вс), пустой если once=true         |
| `once`         | bool     | Единоразовый будильник                                  |
| `sound_source` | string   | `"builtin"` или `"file"`                               |
| `sound_name`   | string   | Имя встроенного звука (если `builtin`)                   |
| `sound_file`   | string?  | Путь к пользовательскому файлу (если `file`), null если нет |
| `volume`       | int      | Громкость 0–100                                         |
| `fade_in`      | bool     | Постепенное увеличение громкости                        |
| `snoozed_until`| string?  | Время отложенного срабатывания в формате ISO или null    |

#### 4.5.3. Автосохранение

- При каждом изменении (создание, редактирование, удаление, toggle) — JSON перезаписывается немедленно (или с задержкой 500 мс через debounce).
- Перезапись атомарная: сначала запись во временный файл, затем замена (чтобы избежать повреждения данных).

#### 4.5.4. Загрузка при старте

- При запуске приложения:
  1. Проверить существование `alarms.json`.
  2. Если файл есть — загрузить и распарсить.
  3. Если файла нет — создать пустой список.
  4. Валидация данных: проверка формата полей.
  5. Восстановить snoozed: если `snoozed_until` больше текущего времени — будильник считается активным; если меньше — игнорируется.

- **Тень** у всплывающих окон (alarm window): blur=30, offset=8, opacity=0.4.

## 5. Нефункциональные требования

### 5.1. Технические требования

| Требование                  | Значение                                           |
|-----------------------------|----------------------------------------------------|
| Язык программирования       | Python 3.11+                                       |
| GUI-фреймворк               | PySide6 (только! Не PyQt, не Tkinter)              |
| Минимум зависимостей        | runtime: PySide6; dev: PyInstaller                 |
| Сборка                      | Один .exe файл через PyInstaller                   |
| Код                         | На английском (переменные, функции, классы, комментарии) |
| UI-текст                    | На русском (все надписи, подписи, сообщения)       |
| Платформа                   | Windows 10/11 (64-bit)                             |
| Архитектура                 | x86_64                                             |

### 5.2. Требования к производительности

- **Потребление памяти**: не более 100 MB в фоне (в трее).
- **Потребление CPU**: 0% в простое, до 5% при срабатывании.
- **Запуск**: не более 3 секунд на холодный старт.
- **Проверка будильников**: каждые 30 секунд в фоновом потоке (QTimer).

### 5.3. Требования к безопасности и надёжности

- При аварийном завершении (kill процесса) — данные не должны теряться (автосохранение после каждого изменения).
- JSON-файл не должен повреждаться при конкурентной записи (запись через temp + rename).
- Будильник **обязан сработать** даже если:
  - Компьютер вышел из сна (sleep/hibernate).
  - Приложение было свёрнуто в трей.
  - Пользователь неактивен (экран заблокирован).
- Обработка некорректных JSON-данных: игнорировать повреждённые записи, логировать ошибку.

### 5.4. Требования к коду

- **Архитектура**: Model-View-Controller (MVC) или Model-View-ViewModel (MVVM) в рамках PySide6.
- Разделение на слои:
  - **Model**: бизнес-логика, работа с JSON, таймеры.
  - **View**: Qt-виджеты, стили, анимации.
  - **Controller/ViewModel**: связь модели и представления, обработка событий.
- Типизация: аннотации типов (type hints) для всех функций и методов.
- Документация: docstrings для классов и публичных методов (Google-style).
- Линтер: flake8 или black (на усмотрение разработчика).

### 5.5. Требования к сборке

- **PyInstaller**: single-file .exe с иконкой (через `--icon=icon.ico`).

### 12.2. `models/alarm_model.py` — модель данных

```python
"""
Alarm data model.
"""

from __future__ import annotations

import uuid
from dataclasses import dataclass, field, asdict
from typing import Optional


@dataclass
class Alarm:
    """Represents a single alarm."""
    time: str  # "HH:MM"
    enabled: bool = True
    title: str = ""
    days: list[int] = field(default_factory=list)  # 1=Mon ... 7=Sun
    once: bool = True
    sound_source: str = "builtin"  # "builtin" | "file"
    sound_name: str = "classic"
    sound_file: Optional[str] = None
    volume: int = 80
    fade_in: bool = False
    snoozed_until: Optional[str] = None
    id: str = field(default_factory=lambda: str(uuid.uuid4()))

    def to_dict(self) -> dict:
        return asdict(self)

    @classmethod
    def from_dict(cls, data: dict) -> "Alarm":
        return cls(**data)
```
